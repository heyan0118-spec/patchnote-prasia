# 수집 전략

## 목표
공식 업데이트 페이지에서 패치노트를 안정적으로 누적 수집하고, 같은 문서를 중복 저장하지 않으며, 이후 질의 품질에 필요한 최소 정규화까지 수행한다.

## 기본 흐름
1. 목록 페이지 요청
2. 패치노트 링크/제목/날짜 추출
3. 저장된 URL과 비교해 신규 여부 판별
4. 신규 또는 내용 변경 문서만 상세 수집
5. 본문 정제 후 문단/항목 단위 청크화
6. 토픽 태깅 + 정책 플래그 저장
7. 임베딩 생성 및 인덱스 반영

## 중복 방지
- 1차 키: `url`
- 2차 확인: `content_hash`
- 같은 URL인데 본문 해시가 달라지면 `updated`로 기록

## 청크 전략
- 너무 길게 자르지 말고, 제목/소제목/목록 기준으로 의미 단위 유지
- 권장: 300~800 토큰 수준
- 이벤트 보상표, 클래스 변경 항목은 별도 청크가 유리

## 태깅 규칙
최소한 아래 범주부터 시작:
- `system`
- `class`
- `event`
- `world_open`
- `balance`
- `item`
- `maintenance`

정책:
- 기본값: `prefer_latest=true`
- 예외: `event`, `class`, `world_open` → `preserve_history=true`

## 운영 메모
- 주 1회 수요일 오전 실행이면 충분
- 첫 배치에서는 과거 문서를 가능한 범위에서 백필
- HTML 구조가 바뀔 수 있으니 selector 의존을 최소화하고, 실패 로그를 남길 것
